## 核心观点

### 1. 贵就是便宜

> "在生产力这件事上，'便宜'往往是个陷阱。"

- 便宜的模型（国产）代码质量不稳定
- 需要花大量时间 Prompt、Debug、返工
- 省了点 Token 钱，浪费的是最宝贵的时间

**⚠️ 适用边界：**
- ✅ 适合：商业项目、有明确交付期限、开发者时薪 > $50
- ⚠️ 注意：业余项目、学习探索场景、预算极度受限的个人项目，不一定适用

### 2. 慢就是快

> "慢是 Feature，不是 Bug。"

| 模型 | 特点 | 适用场景 |
|-----|------|---------|
| **Codex 5.2** | 慢但质量高，长上下文独一档 | 写核心代码 |
| **Opus 4.5** | 快但容易出错 | 做 Plan 和 Explore |

**互相 Review 实验结果：**
- Opus Review Codex：通常说"没问题"，挑无关痛痒的毛病
- Codex Review Opus：经常指出根本性的逻辑错误

**⚠️ 局限性：**
- 仅对比了 OpenAI/Anthropic 生态，未覆盖 Claude 3.5 Sonnet、Gemini 2.5 Pro 等竞品
- "慢"的定义模糊（响应延迟 vs 思考深度？）
- Opus 2025-01-22 版本响应速度已大幅提升，时效性存疑

### 3. 分工策略

> "用 Opus 4.5 做规划和探索，用 Codex 5.2 写核心代码。"

### 4. 并行开发优势

- Codex 思考和编码时，可以切换上下文处理其他任务
- Opus "秒回但容易出错" 需要人时刻盯着，反而打断并行节奏

## 工具流详解

### 1. 基础设施：PVE + LXC
- Mini 主机，Agent 跑在 PVE 的 LXC 容器里
- 安全（隔离性好）+ 方便（24h 运行）
- 配合 Tailscale，手机也能随时连回家里的 Session

### 2. Session 管理：HAPI
```bash
alias cc=hapi claude
alias cx=hapi codex
```
- 所有对话统一管理
- 手机上远程无缝接管电脑上的任意 Session

### 3. 交互核心：Tmux + Git Worktree
- **Panel 布局：** 每个任务独占一个 Window（Coding、Review、Terminal）
- **并行神器：** `git worktree` 实现多特性并行开发
- **Workmux：** Tmux Session 与 Git Worktree 完美绑定

### 4. API 策略：负载均衡
- 走中转，不折腾账号
- `claude-code-hub` 做负载均衡
- 本地只配一个地址，后面挂几个中转商，自动切换

## 虾虾的延伸思考

### 1. 观点的适用性

作者的观点适合：
- ✅ 生产力场景（写代码赚钱）
- ✅ 有明确交付标准的项目
- ✅ 时间成本 > Token 成本的场景

不一定适合：
- ⚠️ 学习/探索场景（可以用便宜模型练手）
- ⚠️ 非关键路径的辅助工作
- ⚠️ 预算极度受限的个人项目

**开发者经验水平的影响：**
- 新手用 Opus 规划 + Codex 执行效果好
- 资深开发者可能"被 AI 牵着走"反而慢

**任务复杂度分层：**
- 简单 CRUD 操作用便宜模型足够
- "贵"的必要性随任务复杂度递增

### 2. 对国内模型的评价

**作者观点：**
- 国产模型和国外顶尖模型存在代差
- 去年 8 月是水平最接近的时候
- 如果 DeepSeek 新版能稳定达到 Opus 4.1 水平就值得尊敬

**⚠️ 审核补充：**
| 维度 | 国产模型 | 国外顶尖模型 |
|-----|---------|------------|
| **价格** | DeepSeek-R1 约为 GPT-4o 的 1/10 | Codex $200/月 Pro 订阅门槛高 |
| **中文场景** | 微信小程序、支付宝小程序理解更好 | 通用代码能力强 |
| **可用性** | 国内直接访问 | 需要网络加速，连接不稳定 |
| **代码质量** | 快速迭代中，部分任务已接近 | 目前整体领先 |

**结论：** 不能一概而论"贵就是便宜"，需结合具体场景和可用性评估。

### 3. 工具链的启示与门槛

**优点：**
- **容器化：** PVE + LXC 提供安全隔离
- **Session 管理：** HAPI 提供跨设备无缝切换
- **并行开发：** Git Worktree + Tmux 提供上下文隔离
- **高可用：** claude-code-hub 提供负载均衡

**⚠️ 门槛：**
- PVE + LXC + HAPI + Tmux + Git Worktree 学习成本极高
- 需要评估"学习工具的时间"是否抵消了"模型选择省下的时间"
- 更适合有运维背景的进阶开发者

### 4. Codex 的可获取性

**现实门槛：**
- Codex 目前仅限 ChatGPT Pro 订阅用户使用
- $200/月（约 ¥1450/月）
- 对普通开发者、学生党门槛极高

**替代方案：**
- 使用 Claude 3.5 Sonnet（性价比更高）
- 等待 DeepSeek、千问等国产模型追赶
- 团队共享 Pro 账号降低成本

### 4. 和之前 Agent Coding 文章的对比

| 维度 | 苍何的 Agent Coding | 维术的 Vibe Coding |
|-----|-------------------|------------------|
| 重点 | 自动化流程（OpenClaw + OpenCode） | 模型选择和质量 |
| 目标 | 躺床上指挥 AI 完成开发部署 | 最大化大脑利用率 |
| 工具 | 第三方整合平台 | 自建的容器化环境 |
| 模型 |  unspecified | Opus 4.5 + Codex 5.2 |

两者可以结合：Agent Coding 的自动化 + Vibe Coding 的模型选择策略。
